MODULO DE INCIDENCIAS — MANUAL DE USO
======================================

REQUISITOS
----------
- Python 3.11 o superior
- pip

INSTALACION
-----------
1. Abrir una terminal en la carpeta del proyecto:

       cd /ruta/al/proyecto/Modulo-de-Incidencias

2. Crear un entorno virtual (recomendado):

       python -m venv venv
       source venv/bin/activate          # Linux / macOS
       venv\Scripts\activate             # Windows

3. Instalar dependencias:

       pip install flask flask-sqlalchemy python-dotenv werkzeug

ARRANCAR EL PROGRAMA
--------------------
Desde la carpeta raiz del proyecto ejecuta:

       python Incidencia/app.py

La aplicacion arranca en:  http://127.0.0.1:8080

Al iniciar por primera vez se crea automaticamente la base de datos SQLite
(Incidencia/incidencias.db) y un usuario administrador con las credenciales:

       Usuario:    admin
       Contrasena: admin123

USAR LA APLICACION
------------------
1. Abrir el navegador en http://127.0.0.1:8080
2. Iniciar sesion con las credenciales anteriores.
3. Desde el panel principal puedes:
   - Ver el listado de incidencias registradas.
   - Crear una nueva incidencia (boton "Registrar incidencia").
   - Ver el detalle de una incidencia (boton "Ver").
   - Editar una incidencia existente (boton "Editar").
   - Eliminar una incidencia (boton "Eliminar").

CAMPOS DEL FORMULARIO
---------------------
  Titulo      : 5–150 caracteres. Debe comenzar con mayuscula,
                tener al menos 2 palabras y no mas de 10.
  Descripcion : 10–2000 caracteres. Debe comenzar con mayuscula
                y contener al menos 3 palabras.
  Responsable : 3–120 caracteres. Solo letras, numeros, espacio,
                punto, guion y guion bajo. Maximo 4 palabras.
  Estado      : Abierta | En progreso | Resuelta | Cerrada
  Prioridad   : Baja | Media | Alta | Critica

EJECUTAR LOS TESTS
------------------
Desde la subcarpeta Incidencia/:

       cd Incidencia
       python -m pytest tests/test_app.py -v

Para ver solo el resumen:

       python -m pytest tests/test_app.py

Se ejecutan 101 pruebas unitarias e integracion que cubren:
  - Autenticacion (login, logout, CSRF, sesion)
  - CRUD completo de incidencias
  - Todas las reglas de validacion de campos
  - Casos borde (valores minimos, maximos, vacios)
  - Flujos de error (incidencia inexistente, duplicados)
  - Tests unitarios directos de funciones de validacion

USO CON POSTGRESQL (opcional)
------------------------------
Crea un archivo Incidencia/.env con las variables:

       POSTGRES_USER=postgres
       POSTGRES_PASSWORD=tu_contrasena
       POSTGRES_HOST=localhost
       POSTGRES_PORT=5432
       POSTGRES_DB=incidencias
       SECRET_KEY=clave-secreta-segura

O directamente con DATABASE_URL:

       DATABASE_URL=postgresql+psycopg://usuario:clave@host:5432/db

DETENER EL SERVIDOR
-------------------
Presionar Ctrl+C en la terminal donde corre el servidor.
